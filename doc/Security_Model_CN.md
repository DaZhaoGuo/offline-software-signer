# 钱包架构与安全模型

## 核心声明

Provides a signing isolation boundary conceptually similar to hardware
wallets, but implemented through software-level isolation and execution
constraints.

（本项目提供一种在理念上类似硬件钱包的签名隔离边界，
但该隔离通过软件级隔离与执行约束实现。）

------------------------------------------------------------------------

## 术语澄清

### 离线签名器 与 冷钱包

本项目使用 **离线签名器（Offline Signer）/ 隔离签名环境** 这一术语。

该架构 **并不天然等同于密码学意义上的冷钱包**。

只有在以下条件同时满足时，才可称为冷钱包： -
私钥或助记词从未在联网环境中生成、导入或使用 -
从未在任何热钱包中进行过在线签名或授权

如果密钥材料曾在联网钱包中使用， 本项目
**不再宣称冷钱包级别的安全保证**。

------------------------------------------------------------------------

## 签名协议正确性（直白解释）

系统安全取决于：
**签名器看到的数据，是否完整、准确且无歧义地反映了用户真实的签名意图。**

包括但不限于： - 签名内容与用户意图完全一致 - 不进行盲签名 -
明确的消息结构与域分离 - 防止交易替换或参数注入

------------------------------------------------------------------------

## 不依赖硬件信任根

本架构不依赖任何硬件信任根，例如： - Secure Element - TPM - TEE
等厂商强制信任环境

所有安全假设均为**显式、可由软件审计的假设**。

------------------------------------------------------------------------

## 逻辑离线设计说明

签名器可以运行在具备联网能力的设备上。

但： - 不申请网络权限 - 不包含任何网络相关代码 -
不暴露远程或进程间通信接口

因此其属于： **逻辑离线、网络隔离的签名模块**，
而非物理意义上的完全断网设备。

------------------------------------------------------------------------

## 安全假设

1.  签名器运行环境未被操作系统级别攻破
2.  应用本身及运行时未被篡改
3.  网络隔离策略真实有效
4.  签名请求真实反映用户意图

------------------------------------------------------------------------

## 安全边界与限制

-   不防御物理设备被攻破的场景
-   不抵御高级侧信道攻击
-   无法在恶意操作系统下保证安全
-   不宣称与硬件钱包或安全芯片等价

------------------------------------------------------------------------

## 设计哲学

本项目强调： - 安全边界清晰 - 威胁模型诚实 -
可审计性优先于营销式安全宣传
